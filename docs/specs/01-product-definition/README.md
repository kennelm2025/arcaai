## What this specification will contain

When complete, this specification will answer:

- **What ArcaAI is** — one-paragraph positioning
- **What problem it solves** — for whom, with what shape
- **Who it is for** — primary target segment, secondary segments, explicit non-targets
- **What outcomes it delivers** — the eleven use cases, framed as value not features
- **What it is not** — and who it is not for
- **Why now** — market timing, technology readiness, regulatory context
- **Success criteria** — how a bank knows ArcaAI worked

## Why this is spec 01

This is the bank's first read. Every other specification is shaped by, and references, the positioning language established here.

## Open strategic questions

Two strategic questions are deferred from the foundational ADR work and must be resolved within this specification:

1. **Target market segment** — mid/small UK banks is the working assumption; needs proper segment analysis and ratification (will become ADR-0004)
2. **Data strategy details** — which public datasets, which synthetic data sources, what claims can be made about reference model performance (will become ADR-0005)
