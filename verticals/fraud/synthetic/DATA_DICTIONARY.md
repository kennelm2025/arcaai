# Fraud synthetic dataset — data dictionary (B2)

Deterministic synthetic card-transaction data for the fraud vertical reference
implementation. Generated by `verticals/fraud/synthetic/generator.py` from
`config.yaml` (seed-pinned, DVC-versioned). **Synthetic means synthetic:** no
real customer data, and model results on this data demonstrate the pipeline,
not production performance.

## transactions.parquet (~957k rows, 17 months, default config)

| Column | Type | Description |
|---|---|---|
| transaction_id | str | `T{seq}` — unique, assigned after chronological sort |
| timestamp | datetime | Transaction time, 2025-01-01 → 2026-05-31 |
| customer_id | str | FK → customers.parquet |
| merchant_id | str | FK → merchants.parquet |
| merchant_category | str | 12 categories (grocery … jewellery) |
| merchant_country | str | ISO-ish; GB dominant, 8 intl |
| amount | float | GBP; lognormal per customer × category multiplier; Dec uplift |
| currency | str | Always GBP (R6) |
| channel | str | pos / contactless / online / atm |
| device_id | str | Customer primary device (online), POS-TERMINAL, or attacker `DX*` |
| is_international | bool | Merchant non-GB or travel behaviour |
| customer_home_region | str | UK region |
| is_fraud | int | 0/1 — independent binary outcome per txn (R10: no group normalisation) |
| fraud_pattern | str | none / spree / testing / takeover / first_party |
| label_available_date | datetime | When the label becomes known: fraud = txn + Γ(mean 18d, cap 90d); legit = txn + 45d settle. **Training pipelines must respect this** (anti-leakage rule 3). |

## Fraud patterns (default config → ~0.5% fraud by count)

| Pattern | Mechanic | Signature the model can learn |
|---|---|---|
| spree (~2.5k rows) | Stolen card: 5–15 escalating txns in minutes–hours at high-risk merchants, attacker device, often night/intl | velocity, amount escalation, device novelty, category shift |
| testing (~1.6k) | 6–20 micro-amounts (<£2) online in quick succession, 70% followed by a £200–900 strike | micro-amount bursts, single new device |
| takeover (~0.6k) | New device, online, 1.5–4× customer mean amounts over 1–4 days | device novelty + amount shift sustained |
| first_party (~0.4k) | Genuine high-amount online txns relabelled — **deliberately near-indistinguishable** | almost none — keeps achievable AUC honest |

## Honest limitations

- Fraud rate ~0.5% by count is **above** real-world card fraud (~0.1%) — a
  deliberate design choice so Phase-1-scale data carries enough positives;
  class-weighting decisions belong to B4 and must note this.
- One card/device per customer; no issuer declines, reversals, or partial
  auths; amounts uncorrelated with income; geography is coarse (region level).
- Seasonality limited to a December volume/amount lift — drift across
  walk-forward periods is therefore mild; real data will be harsher.
- Patterns are injected, hence cleaner than reality even with the first-party
  hard cases. G2's baseline and G5's MVM gates interpret results accordingly.

## companions
